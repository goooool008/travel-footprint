## 一、项目简介

**旅行足迹** 是一个纯前端网页应用，展示 3D 地球模型，让用户标记自己去过的国家和城市。支持按年份筛选、统计旅行数据。无需后端，每个用户的数据存储在各自浏览器的 localStorage 中。

## 三、工作规范

### 3.1 开发原则
- **小步迭代**：每次只推进一个阶段，或一个阶段中的 1-2 个任务
- **先确认、再动手**：每个阶段开始前与用户沟通，完成后请用户验收
- **不跳步**：严格按照 `execution-plan.md` 中的阶段顺序推进
- **可见可测**：每个阶段结束时必须有可运行的成果

### 3.2 工作流程
1. 查看 `execution-plan.md`，确认当前处于哪个阶段
2. 阅读当前阶段的任务清单
3. 与用户确认开始该阶段
4. 逐个完成任务，每完成一个关键任务告知用户
5. 阶段完成后请用户打开页面验证
6. 更新 `execution-plan.md` 中的阶段状态

### 3.3 技术约束
- **不引入框架**：使用原生 HTML + CSS + JavaScript
- **CDN 加载依赖**：不安装 npm 包，不创建 package.json
- **所有依赖从 CDN 加载**：Three.js、Globe.GL 等
- **数据只存本地**：不做任何网络请求上传用户数据
- **文件路径使用绝对路径**：项目根目录 `D:/旅行足迹/`

### 3.4 沟通规范
- 需求不明确时参考 `docs/requirements.md`，仍有疑问再问用户
- 设计方案参考 `docs/design-spec.md`
- 技术方案参考 `docs/tech-spec.md`
- 对需求或设计有改进建议时，先提出讨论，不自行修改
