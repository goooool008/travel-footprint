# 📋 旅行足迹 - 执行步骤

> 版本：v1.0 | 最后更新：2026-06-24

---

## 开发原则

1. **小步迭代**：每个阶段只做一件事，确认可用后再进入下一阶段
2. **可见可测**：每个阶段结束时有可运行、可演示的成果
3. **先骨架后血肉**：先跑通核心流程，再加细节功能
4. **不预优化**：先让功能正确，再做性能优化

---

## 阶段划分

### 🟢 第一阶段：项目骨架（预计 1 轮对话）

**目标**：HTML 页面能打开，3D 地球能显示、能旋转缩放

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 1.1 | 创建 `index.html` 基础结构 | 页面可打开 |
| 1.2 | 引入 Three.js + Globe.GL CDN | 无控制台报错 |
| 1.3 | 初始化 3D 地球（默认暗色） | 地球显示，鼠标可拖拽旋转/缩放 |
| 1.4 | 加载 GeoJSON 国家边界数据 | 国家轮廓可见 |
| 1.5 | 基础 CSS 星空背景 | 深色星空背景 |

**验收**：打开页面 → 看到星空背景上一个可交互的 3D 地球

### 🟢 第二阶段：核心交互（预计 1-2 轮对话）

**目标**：能点击国家标记"去过"，去过的地方变亮

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 2.1 | 实现国家点击事件（Raycaster） | 点击国家有响应 |
| 2.2 | 全局视角下：去过国家高亮，未去过暗色 | 颜色正确区分 |
| 2.3 | localStorage 读写数据 | 刷新后数据保留 |
| 2.4 | 基础统计：已点亮国家数显示在右上角 | 数字正确更新 |

**验收**：点击一个国家 → 变亮 → 刷新不丢 → 统计数字 +1

### 🟢 第三阶段：搜索与城市（预计 1-2 轮对话）

**目标**：搜索框可用，点击国家可进入城市视角

| 任务 | 产出 | 验收标准 |
