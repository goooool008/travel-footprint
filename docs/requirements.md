# 🌍 旅行足迹 - 需求文档

> 版本：v1.0 | 最后更新：2026-06-24

---

## 一、项目概述

一个**纯前端网页应用**，展示 3D 地球模型，让用户手动标记自己去过的国家和城市，记录旅行足迹。支持按年份筛选，统计旅行数据。

---

## 二、目标用户

- 任何想记录旅行足迹的普通人
- 无需注册登录，打开即用
- 数据存在自己的浏览器中，互不干扰

---

## 三、功能需求

### 3.1 3D 地球模型
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-01 | 展示 3D 地球，支持鼠标拖拽旋转、滚轮缩放 | P0 |
| F-02 | 去过的国家在全球视角下显示为**亮色/高亮**，未去过的显示为暗色 | P0 |
| F-03 | 点击一个国家 → 放大进入该国，显示城市级别的光点 | P0 |
| F-04 | 在国家内部视角，去过的城市高亮，未去过的暗色 | P0 |

### 3.2 地点选择
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-05 | 在地球上直接点击国家/城市来标记"去过" | P0 |
| F-06 | 搜索框：输入国家/城市名称搜索并添加 | P0 |
| F-07 | 已标记列表：展示所有去过的地方，可删除/修改 | P1 |

### 3.3 时间维度
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-08 | 每次标记时可记录旅行时间（年月日） | P0 |
| F-09 | 按年份筛选："只看 2025 年去过哪里" | P0 |
| F-10 | "一生总览"模式：显示所有年份足迹 | P0 |

### 3.4 统计面板
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-11 | 统计点亮的国家数量 | P0 |
| F-12 | 统计点亮的城市数量 | P0 |
| F-13 | 统计累计覆盖面积（去过国家面积之和） | P1 |
| F-14 | 计算并显示"去过的最远地方"（以常驻城市为起点） | P1 |

### 3.5 国家信息
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-15 | 点进一个国家时，侧边栏显示该国基本信息和介绍 | P1 |

### 3.6 数据管理
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-16 | 数据存储在浏览器 localStorage | P0 |
| F-17 | 导出数据为 JSON 文件备份 | P1 |
| F-18 | 导入备份文件恢复数据 | P1 |

### 3.7 用户设置
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-19 | 在地图上设置"常驻城市"（家乡），作为最远距离起点 | P1 |
| F-20 | 常驻城市在地球上特殊标记显示 | P2 |

---

## 四、UI 设计要求

| 编号 | 需求 |
|------|------|
| UI-01 | 整体风格简洁明了 |
| UI-02 | 背景为星空 |
| UI-03 | 侧边栏或浮动面板展示统计信息和筛选控件 |
| UI-04 | 搜索结果下拉列表，快速选择 |
| UI-05 | 响应式设计（桌面端优先） |

---

## 五、非功能需求

| 编号 | 需求 |
|------|------|
| NFR-01 | Windows 浏览器兼容（Chrome / Edge / Firefox） |
| NFR-02 | 首次加载 < 5 秒 |
| NFR-03 | 3D 渲染流畅（60fps 目标） |
| NFR-04 | 所有数据本地存储，不上传服务器 |
| NFR-05 | 部署到公网，生成分享链接 |
